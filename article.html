WePhone - 基于 WebRTC 的实时音视频通话应用

WebRTC Node.js Socket.io Docker

在现代 Web 应用开发中，实时音视频通信（Real-Time Communication）的需求日益增长。为了深入探索底层技术原理，并解决跨设备低延迟通信的痛点，我在 2026 年初开发了 WePhone——一款基于 WebRTC 协议实现的点对点（P2P）实时音视频通话应用。

核心功能特性

  • 多端互通： 运用 WebRTC 原生 API，支持主流浏览器和移动设备间的音视频无缝对接。
  • 超低延迟通信： 建立 P2P 专用通道，音视频流数据不经服务器中转直接送达对端。
  • 动态编解码优化： 自适应切换分辨率与码率，实现在弱网环境下的体验最佳化。

系统架构设计

项目的整体架构分为前端客户端、信令服务器（Signaling Server）以及穿透中继服务器（STUN/TURN Server）。

信令交换过程 (Signaling)

WebRTC 本身不包含信令的标准实现，因此我选择了 Node.js + Socket.io 来构建高并发并具备容错能力的信令服务器。主要处理：

  • 房间管理机制： 客户端通过指定房间号加入逻辑隔离的会话空间。
  • SDP 交换： 处理 Offer/Answer 模式下的媒体能力协商。
  • ICE 候选者转发： 在建立 P2P 通信前，及时同步客户端收集到的网络连通性信息。

代码实现片段

以下是服务端处理信令交换的核心逻辑示例：

// Socket.io 信令交换处理
io.on('connection', socket => {
  socket.on('join-room', (roomId, userId) => {
    socket.join(roomId);
    socket.to(roomId).emit('user-connected', userId);
    
    // 监听 Offer / Answer
    socket.on('offer', (payload) => {
      io.to(payload.target).emit('offer', payload);
    });

    socket.on('answer', (payload) => {
      io.to(payload.target).emit('answer', payload);
    });

    // 监听 ICE Candidate 转发
    socket.on('ice-candidate', (payload) => {
      io.to(payload.target).emit('ice-candidate', payload);
    });
    
    socket.on('disconnect', () => {
      socket.to(roomId).emit('user-disconnected', userId);
    });
  });
});

部署方案 (Docker Compose)

为了免除繁杂的环境配置，提升部署效率，项目采用了容器化部署方案。通过编写 Dockerfile 将 Node.js 服务封装，配合 Docker Compose 一键拉起应用层以及 Coturn (TURN/STUN 服务)，实现了高度的自动化：

version: '3.8'
services:
  wephone-signaling:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
    restart: always

  turn-server:
    image: coturn/coturn
    ports:
      - "3478:3478/udp"
    command: >
      -n --log-file=stdout --min-port=49152 --max-port=65535 
      --user=gzy:wephone --realm=gede.asia
